## Задание 2 — Отладка приложения «Фибоначчи»

**Ссылка на туториал:** https://learn.microsoft.com/ru-ru/training/modules/dotnet-debug-visual-studio/4-use-visual-studio-debugger  
**Ветка репозитория:** `Task2`

### Описание

Консольное приложение генерирует последовательность Фибоначчи с помощью `List<int>`.  
При первом запуске программа завершается с исключением.

### Ошибки, обнаруженные при отладке

**Ошибка 1 — `IndexOutOfRangeException` при первой итерации цикла**

- **Где:** метод `GetFibonacciNumbers`, строка `var previous2 = list[list.Count - 2]`
- **Причина:** список инициализируется одним элементом `{ 1 }`. На первой итерации `list.Count == 1`, поэтому `list.Count - 2 == -1` — недопустимый индекс.
- **Исправление:** инициализировать список двумя элементами: `{ 1, 1 }`.

**Как был найден баг:**  
Запустил приложение с отладчиком (F5). Visual Studio остановился на строке обращения к индексу и отобразил `IndexOutOfRangeException` в окне Exception Helper. В окне «Локальные» видно, что `list.Count == 1`, из-за чего `list.Count - 2 = -1`.

### Использованные способы отладки

| Инструмент | Применение |
|---|---|
| Точка останова (F9) | Поставлена на строке `list.Add(previous + previous2)` для проверки значений перед добавлением |
| Окно «Локальные» | Отслеживал значения `i`, `previous`, `previous2`, `list.Count` на каждой итерации |
| Подсказки по данным (DataTips) | Навёл мышь на переменную `list` — увидел её текущее содержимое в виде дерева |
| Шаг с заходом — Step Into (F11) | Вошёл в метод `GetFibonacciNumbers` для пошагового выполнения |
| Шаг — Step Over (F10) | Перебирал итерации цикла, не заходя внутрь вызовов |
| Exception Helper | Автоматически показал место и тип исключения при первом запуске с багом |

## Задание 3 — Отладка приложения «Галактики»

**Ссылка на туториал:** https://learn.microsoft.com/ru-ru/visualstudio/debugger/debugging-absolute-beginners?view=vs-2022&tabs=csharp  
**Ветка репозитория:** `Task3`

### Описание

Консольное приложение выводит список галактик с их типами (Spiral, Elliptical, Irregular, Lenticular).  
При запуске одна галактика («Small Magellanic Cloud») отображается с пустым типом вместо «Irregular».

### Ошибка, обнаруженная при отладке

**Ошибка — неправильный символ в `switch`-ветке класса `GType`**

- **Где:** класс `GType`, конструктор, `switch`-выражение
- **Причина:** ветка для типа `Irregular` проверяет символ `'l'` (строчная латинская L), тогда как при создании галактики передаётся `'I'` (заглавная I). Символы визуально похожи, но не равны. В результате ни одна ветка `switch` не срабатывает, `MyGType` остаётся `null`, и в консоль выводится пустая строка.
- **Исправление:** заменить `case 'l':` на `case 'I':`.

**Как был найден баг:**  
Поставил точку останова в конструкторе `GType`. В окне «Локальные» видно, что параметр `type` содержит значение `I`, а при выполнении `switch` ни одна ветка не совпадает. После выхода из конструктора в окне «Авто» свойство `MyGType` равно `null`. Это и является причиной пустого вывода.

### Использованные способы отладки

| Инструмент | Применение |
|---|---|
| Точка останова (F9) | Поставлена в конструкторе `GType` и в `foreach`-цикле `IterateThroughList` |
| Окно «Локальные» | Проверял значение параметра `type` и свойства `MyGType` в конструкторе |
| Окно «Авто» (Autos) | Автоматически показывало переменные текущей и предыдущей строки — позволило мгновенно увидеть `null` в `MyGType` |
| Стек вызовов (Call Stack) | Отследил цепочку: `Main → IterateThroughList → GType..ctor` |
| Подсказки по данным (DataTips) | Навёл мышь на объект `theGalaxy` — увидел все его свойства, включая пустой `GalaxyType` |
| Шаг с заходом (F11) | Вошёл в конструктор `GType` для проверки `switch` |

## Задание 4 — Отладка приложения «Буквы»

**Ссылка на туториал:** https://learn.microsoft.com/ru-ru/visualstudio/get-started/csharp/tutorial-debugger?view=vs-2022#create-a-project  
**Ветка репозитория:** `Task4`

### Описание

Консольное приложение последовательно составляет имя `"fred smith"` из массива символов и на каждом шаге вызывает метод `SendMessage`, выводящий накопленную строку и счётчик.  
Программа работает без ошибок и используется как учебный пример для знакомства с инструментами отладчика.

### Что наблюдалось при отладке

- Поставил точку останова на строке `name += letters[i]`.
- На каждой остановке отладчика в окне «Локальные» видно, как переменная `name` накапливает символы: `"f"` → `"fr"` → `"fre"` → ... → `"fred smith"`.
- В окне «Контрольные значения» (Watch) добавил переменные `name`, `i`, `a[i]` — можно наблюдать их изменение в реальном времени без повторных точек останова.
- Использовал условную точку останова: `i == 3` — отладчик остановился только на 4-й итерации (символ `'d'`), а не на каждой.

### Использованные способы отладки

| Инструмент | Применение |
|---|---|
| Точка останова (F9) | На строке `name += letters[i]` — для наблюдения за накоплением строки |
| Условная точка останова | Условие `i == 3` — остановка только на нужной итерации |
| Окно «Контрольные значения» (Watch) | Добавлены `name`, `i`, `a[i]` для постоянного мониторинга |
| Окно «Локальные» | Автоматически отображал все переменные метода `Main` |
| Шаг с заходом (F11) | Вошёл в метод `SendMessage` для проверки формируемой строки |
| Шаг с выходом (Shift+F11) | Быстро вышел из `SendMessage` обратно в `Main` |
| Подсказки по данным (DataTips) | Навёл мышь на `letters[i]` — увидел текущий символ без остановки |

## Вывод по работе

В ходе практической работы были освоены основные инструменты отладчика Microsoft Visual Studio:

- **Точки останова** (обычные, условные, временные) — позволяют приостановить выполнение в нужном месте кода.
- **Окна наблюдения** («Локальные», «Авто», «Контрольные значения») — дают полную картину состояния переменных в любой момент.
- **Пошаговое выполнение** (Step Into, Step Over, Step Out) — помогает точно отследить порядок выполнения кода.
- **Стек вызовов** (Call Stack) — позволяет понять, откуда был вызван текущий метод.
- **DataTips** — быстрый просмотр значений переменных непосредственно в редакторе кода.

Благодаря этим инструментам были обнаружены и устранены ошибки:
1. `IndexOutOfRangeException` в приложении «Фибоначчи» — неправильная инициализация списка.
2. Визуально похожие символы `'l'` и `'I'` в `switch`-конструкции приложения «Галактики».
